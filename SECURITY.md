# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Send an email to **info@vibecodingacademy.nl** with:

1. **Description** — What is the vulnerability?
2. **Impact** — What could an attacker do with this?
3. **Steps to reproduce** — How can we verify the issue?
4. **Affected components** — Which plugin(s) or file(s) are affected?

### What to Expect

| Timeframe | Action |
|-----------|--------|
| **5 business days** | Initial response acknowledging your report |
| **10 business days** | Assessment and next steps communicated |
| **30 days** | Fix released (for confirmed vulnerabilities) |

### Scope

This policy covers:
- All plugins in this repository
- Documentation that could lead to insecure implementations
- The installation script (`install.sh`)

### Out of Scope

- Vulnerabilities in third-party dependencies (report to the dependency maintainer)
- Issues in projects that *use* these plugins (not the plugins themselves)

### Recognition

We appreciate responsible disclosure. With your permission, we'll acknowledge your contribution in the release notes.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x.x   | ✅ Yes    |
| < 1.0   | ❌ No     |

---

Thank you for helping keep this project secure.
